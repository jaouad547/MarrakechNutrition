# Task 04: User Profiles

## Goal
Allow authenticated users to view and edit their personal information and change their password.

## Parent Milestone
[Milestone 2: Authentication](../00-main-plan.md)

## Prerequisites
- [Task 03: Authentication System](../02-authentication/task-03-auth-system.md)

## Acceptance Criteria
- [ ] Authenticated users can access a profile page from the client dashboard
- [ ] Profile page displays current name, email, phone, and address in editable form fields
- [ ] Users can update their personal information and see a success confirmation
- [ ] Validation errors appear if submitted data is invalid
- [ ] Users can change their password by entering current and new password
- [ ] Password change requires confirmation field matching
- [ ] Updated profile information is reflected immediately after saving
- [ ] Users cannot update email to an address already used by another account

## Files to Create / Modify
- Profile page component
- Profile update controller / route
- Password change controller / route
- Navigation link to profile page

2–3 hours
