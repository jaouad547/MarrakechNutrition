# Task 12: Order Confirmation

## Goal
Show a confirmation page after successful order placement with order details and next steps.

## Parent Milestone
[Milestone 5: Order & Checkout](../00-main-plan.md)

## Prerequisites
- [Task 11: Order Processing](../05-checkout/task-11-order-processing.md)

## Acceptance Criteria
- [ ] After placing an order, user is redirected to a confirmation page
- [ ] Confirmation page displays the order number prominently
- [ ] Confirmation page lists all ordered items with names, quantities, and prices
- [ ] Confirmation page shows delivery address and phone number
- [ ] Confirmation page shows total amount and payment method
- [ ] Confirmation page includes estimated delivery timeline message
- [ ] Confirmation page has a button to return to the home page
- [ ] Confirmation page has a button to view order details in client area (for authenticated users)
- [ ] Accessing confirmation page with an invalid or non-existent order ID shows an error
- [ ] Confirmation page content is printer-friendly

## Files to Create / Modify
- Order confirmation page component
- Confirmation controller
- Routes for order confirmation

## Estimated Duration
2–3 hours
